== Description ==
Multisite Manager gives Super Admins centralized control to manage subsites with data comparison, global updates, user management, settings synchronization, reports, analytics, and maintenance across the network.

== Purpose ==
The purpose of this plugin is to provide network admins with a centralized platform to manage content and analyze data across all subsites without the need to visit each subsite individually. It streamlines the process of handling network-wide updates, settings synchronization, and monitoring performance.

== Usage ==
The plugin is designed for use in WordPress multisite networks. Super Admins can access a unified dashboard to control subsites, manage users, and generate network-wide reports.

== Features ==
1. **Centralized Management**: Manage all subsites' content, users, and settings from a single place.
2. **Data Analysis**: Analyze data across subsites with comprehensive reports and analytics.
3. **Global Updates**: Push global updates to all subsites simultaneously.
4. **Settings Synchronization**: Sync settings across all network subsites with ease.
5. **User Management**: Manage users for the entire network from one dashboard.
6. **Reports & Analytics**: Generate and export network-wide reports and monitor performance metrics.
7. **Maintenance Tools**: Perform maintenance tasks such as backups, updates, and cleanups across the network.

== Installation ==
1. Download the plugin from the repository.
2. Upload the `multisite-manager` folder to the `/wp-content/plugins/` directory.
3. Activate the plugin through the 'Plugins' menu in WordPress.
4. Access the Multisite Manager dashboard from the network admin panel.
